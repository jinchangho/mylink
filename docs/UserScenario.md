# [User Scenario] 마이링크 (MyLink)

이 문서는 "마이링크" 서비스의 주요 사용자 그룹별 이용 시나리오를 정의합니다.

---

## 1. 페르소나 정의
- **소유자 (Owner)**: 자신의 SNS 링크를 한곳에 모으고 싶은 크리에이터 혹은 개인. 구글 로그인을 통해 페이지를 관리함.
- **방문자 (Visitor)**: 소유자의 SNS 프로필 링크 등을 통해 마이링크 페이지에 접속하여 정보를 확인하는 사람.

---

## 2. 시나리오 A: 소유자의 서비스 이용 (관리 및 설정)

### 2.1 가입 및 관리자 접속
1. 사용자는 '마이링크' 메인 페이지에 접속한다.
2. **[구글 로그인]** 버튼을 클릭하여 소셜 연동을 진행한다.
3. 로그인 성공 후, 자신의 고유 URL로 관리자 대시보드에 접근한다.

### 2.2 링크 추가 및 관리
1. 사용자는 대시보드에서 **[새 링크 추가]** 버튼을 클릭한다.
2. 링크의 **제목**(예: "내 유튜브 채널")과 **URL**(예: `youtube.com/@...`)을 입력한다.
3. 입력 즉시 구글 파비콘 API를 통해 해당 사이트의 아이콘이 자동으로 생성되는 것을 확인한다.
4. 필요 없는 링크가 있다면 **[삭제]** 버튼을 통해 목록에서 제거한다.

---

## 3. 시나리오 B: 방문자의 페이지 방문 (열람 및 클릭)

### 3.1 페이지 접속
1. 방문자는 소유자의 인스타그램 프로필 상단에 걸린 `mylink.com/username` 링크를 클릭한다.
2. 모바일 브라우저를 통해 소유자의 마이링크 페이지가 로드된다.

### 3.2 링크 확인 및 이동
1. 방문자는 나열된 링크 리스트를 살펴본다.
2. 각 링크 왼쪽에 표시된 사이트 아이콘(파비콘)을 통해 어떤 서비스인지 직관적으로 파악한다.
3. 원하는 링크 버튼을 클릭한다.
4. 새 창(또는 현재 창)으로 해당 웹사이트가 열리며 이동한다.

---

## 4. 예외 시나리오 (Exception Flow)

### 4.1 존재하지 않는 사용자 페이지 접속
- 방문자가 잘못된 URL(예: `mylink.com/invalid_user`)로 접속한 경우.
- 시스템은 "존재하지 않는 사용자입니다"라는 안내와 함께 메인 페이지로 이동하는 버튼을 노출한다.
